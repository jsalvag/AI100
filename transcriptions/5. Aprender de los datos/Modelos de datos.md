- If you work with a machine learning team,
one phrase you'll hear
is "We're working to improve the data model."
In fact, these teams usually put a very high priority
on working to improve the data model.
So what is a data model?
The best way to think of a data model
is an abstraction of what the AI system
learned from all that training and data.
That's why the data model is never finished.
A good machine learning team
will push their model to become more accurate.
So if it's 99% accurate,
the team will push it to become 99.9% accurate,
then 99.999% accurate.
The system needs to continue to learn
until it almost always guesses correctly.
It might sound strange, but you also use data models.
When you get behind the wheel of a friend's car,
you know how to start the engine,
even though you've never driven the car before.
If you see a bird in the sky, you'll recognize it as a bird,
even if you've never seen the species before.
That's because, throughout your life,
you've created data models to help you understand the world.
You've taken a lot of data,
and then you have extracted it down to core concepts.
That way, you can apply these concepts to something new.
The more data you've processed,
the more accurate you'll be at making guesses.
So if you've seen thousands of species of birds,
there's a pretty good chance
that you'll guess the correct species.
The more cars you've driven,
the better you'll be at driving a new car.
In a sense, your data model is what you've learned
from all that training and data.
Data models are important to understand
when comparing predictive
and generative artificial intelligence.
Data models are mostly used for predictive AI.
In fact, most of the machine learning algorithms,
that you'll see later, are used for data models,
so you'll only use them to make predictions.
That's because predictive systems
are trying to get better at making specific guesses.
You'll be creating a data model
to understand that one data set.
It's the same way that you would use
a different data model to drive a car
than would use to identify a bird.
Each of these requires training on a different model.
So, if you drive a lot of cars,
you won't be any better at identifying birds.
You need to train yourself on bird species,
but that won't help you be a better driver.
But remember that generative AI is much more flexible,
but it also needs more data.
Think about if you ask a generative AI chatbot
a complex question.
Like, you ask it to write a story
about you driving in a sports car
with Big Bird from the show "Sesame Street".
The Gen AI system needs to know a lot about birds,
cars, roads, and even television
to be able to generate that story.
Data models are still the core of AI in most organizations.
That's because predictive AI gives you a lot of power,
but not a lot of flexibility.
Later, you'll see a lot more flexible model that you can use
for newer generative artificial intelligence.
